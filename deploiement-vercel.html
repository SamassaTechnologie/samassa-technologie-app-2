🐱 Méthode GitHub + Vercel (Recommandée)
1
Créer un compte GitHub (gratuit)
Allez sur github.com et créez un compte gratuit avec votre email.
🔗 Lien direct github.com/signup — Choisissez le plan gratuit
2
Créer un nouveau dépôt
Sur GitHub, cliquez le bouton + en haut à droite → "New repository".
Nom : samassa-technologie-pro → Cochez "Public" → Cliquez "Create repository".
3
Uploader vos fichiers sur GitHub
Dans votre nouveau dépôt, cliquez "uploading an existing file" ou "Add file → Upload files".
⚠️ Important : ne pas uploader firebase-config.js ! Ce fichier contient vos identifiants privés. Uploadez tous les autres fichiers du ZIP.
📋 Fichiers à uploader (tous sauf firebase-config.js) : index.html, login.html, facture.html, recu.html, devis.html, intervention.html, recu_cyber.html, facture_cyber.html, style.css, mobile.css, auth.js, sync.js, utils.js, *.js, vercel.json, manifest.json, service-worker.js, logo-*.png/jpg, icon-512x512.png, logo.png, signature.png, cachet.png, watermark.png
✓ Cliquez "Commit changes" pour sauvegarder
4
Connecter Vercel à GitHub
Allez sur vercel.com → "Add New Project" → "Import Git Repository" → Sélectionnez votre dépôt samassa-technologie-pro.
⚙️ Paramètres Vercel à vérifier • Framework Preset : Other (pas Next.js !)
• Root Directory : ./ (laisser vide)
• Build Command : laisser vide
• Output Directory : ./ ou laisser vide
5
Ajouter firebase-config.js via les variables d'environnement
Votre firebase-config.js ne doit pas être sur GitHub. Ajoutez-le sur Vercel après le déploiement.
💡 Solution simple Après le déploiement, allez dans votre projet Vercel → Settings → Environment Variables et ajoutez :

Nom : FIREBASE_CONFIG
Valeur : le contenu de votre firebase-config.js
✅ Déploiement terminé ! Vercel vous donne une URL comme : samassa-technologie-pro.vercel.app
C'est votre application en ligne !
📁 Méthode Glisser-Déposer (La plus simple)
1
Extraire le ZIP sur votre ordinateur
Décompressez le fichier samassa-technologie-pro-v2.9.zip dans un dossier sur votre bureau.
2
Ouvrir Vercel et aller dans les Deployments
Allez sur vercel.com → Connectez-vous → Cliquez "Add New Project".
📋 Cherchez l'option "Deploy without code" Sur la page de nouveau projet, cherchez "Browse" ou "Drop folder" tout en bas.
3
Glisser le dossier entier sur Vercel
Faites glisser le dossier décompressé (pas le ZIP) directement dans la zone de dépôt de Vercel.
⚠️ Le fichier vercel.json est maintenant inclus Il dit à Vercel que c'est un site statique. L'erreur précédente est corrigée.
✅ Vercel déploie automatiquement et vous donne une URL
4
Ajouter firebase-config.js après le déploiement
Une fois déployé, allez dans Settings → Environment Variables et ajoutez votre configuration Firebase.
Alternative plus simple Incluez firebase-config.js dans le dossier avant de déposer. Ce n'est pas idéal pour la sécurité mais fonctionne pour un usage personnel/interne.

📱 Installer comme application sur votre téléphone

Une fois votre site en ligne sur Vercel, installez-le comme une vraie application en suivant ces étapes :

1
Ouvrez votre URL Vercel dans Google Chrome sur Android (ex: samassa-tech.vercel.app)
2
Appuyez sur ⋮ les 3 points en haut à droite de Chrome
3
Appuyez sur "Ajouter à l'écran d'accueil" ou "Installer l'application"
4
Appuyez "Ajouter" → L'icône SAMASSA apparaît sur votre bureau 🎉
5
Sur iPhone : Safari → bouton Partager ⬆ → "Sur l'écran d'accueil"
🌿 Alternative plus simple : Netlify

Si Vercel vous pose des problèmes, Netlify est encore plus simple pour les sites statiques :

1
Allez sur app.netlify.com → Connectez-vous
2
Faites glisser votre dossier décompressé dans la zone "Drag and drop" sur la page Sites
3
✅ C'est tout ! Netlify déploie en 30 secondes et vous donne une URL du type samassa-xxxx.netlify.app
🎯 Netlify est souvent plus simple que Vercel pour les sites purement HTML/CSS/JS Aucun fichier de configuration nécessaire — il reconnaît automatiquement les sites statiques.
